# Roadmap

## Purpose

This roadmap defines the staged path for the behavioral adoption ABM experiment.

The experiment should move from conceptual documentation to an executable Python/Mesa simulation, then to integration with the institutional architecture experiment, and finally to calibration with real-world data.

## Stage 0 — Folder and baseline documentation

Status: started.

Deliverables:

- `README.md`;
- `ROLE_ONTOLOGY.md`;
- `METRICS.md`;
- `MODEL_DESIGN.md`;
- `ROADMAP.md`.

Goal:

```text
Make explicit what the new experiment is, why it exists, which behavioral unknowns it targets,
and how it relates to the existing architecture and adversarial ABM work.
```

## Stage 1 — Minimal behavioral model specification

Deliverables:

## Stage 4 — Integration with institutional architecture experiment

Deliverables:

- behavioral-output JSON schema;
- adapter from behavioral outputs to architecture simulation inputs;
- integrated run example;
- comparison of architecture results under synthetic versus behavior-generated parameters.

Core integration question:

```text
Does the preferred Core v0 architecture remain strong when participation, delegation,
fiscalization, evidence supply, executor participation, and trust are generated by a behavioral model
rather than imposed manually?
```

## Stage 5 — Empirical calibration

Deliverables:

- calibration notes;
- parameter provenance table;
- pilot-data adapter;
- updated scenarios with empirical ranges;
- uncertainty intervals.

Possible calibration sources:

- landing-page tests;
- onboarding prototype analytics;
- survey data;
- interviews;
- small bounded pilot;
- participatory-budgeting participation rates;
- civic-tech adoption data;
- volunteer/fiscalizer recruitment rates;
- evidence-provider recruitment tests;
- delegation experiments.

Calibration discipline:

```text
Mark every parameter as synthetic, assumed, literature-derived, llm-elicited, prototype-derived, or pilot-derived.
```

`llm-elicited` (added 2026-07-06) sits between `assumed` and `prototype-derived`: structured synthetic priors from the planning-behavior-calibration panel, model-dependent and subject to stated-intention bias, never a substitute for human data.

## Stage 6 — Paper-grade replication package

Deliverables:

- frozen scenarios;
- reproducible seeds;
- result tables;
- figure generation scripts;
- method appendix;
- limitations section;
- replication instructions.

Goal:

```text
Make the behavioral experiment publishable or at least externally reviewable.
```

## Immediate next tasks

1. Add scenario schema.
2. Add baseline parameter table.
3. Add state-transition document.
4. Implement first minimal Python/Mesa prototype.
5. Run a small synthetic population.
6. Compare outputs against the parameters previously imposed in the architecture experiment.

## Design rule

> Do not integrate this model into the architecture experiment until it can independently explain its own behavioral outputs.

The behavioral model should first stand alone. Only then should it feed the institutional architecture model.
