# Research Method

## Recursive Hypothesis Refinement

This project uses a method of recursive hypothesis refinement.

The method is simple:

1. Capture an intuition or idea.
2. Convert it into a clear hypothesis.
3. Identify assumptions.
4. Attack the hypothesis with objections and edge cases.
5. Refine the hypothesis.
6. Classify the result.
7. Store what survived in the knowledge base.
8. Record unresolved questions.

## Promotion rule

An idea should not be promoted into the core theory merely because it is interesting. It must survive at least one serious round of critique.

## Session protocol

At the end of each important discussion, produce a distillation containing:

- new concepts;
- new hypotheses;
- modified hypotheses;
- objections;
- unresolved questions;
- candidate text for future essays or papers.

## Roles

- **Conversation / Chat:** laboratory for discovery, critique, and refinement.
- **Repository:** persistent brain of the project.
- **tooling / tooling:** implementation layer for files, structure, automation, and publication workflows.

## Rule of persistence

Any important idea that survives the session should leave the chat and enter the repository the same day.
