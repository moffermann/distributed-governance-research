# Session Distillation Skill

## Purpose

Use this skill at the end of an important research conversation, external review exchange, debate, or conceptual breakthrough.

The goal is to extract durable knowledge from conversation before it is lost in chat history.

## Output locations

Session distillations usually go in:

- `knowledge/YYYY-MM-DD-short-session-name.md`
- or `knowledge/##_SESSION_NAME.md` for early foundational sessions.

If new hypotheses emerge, create files in:

- `knowledge/hypotheses/`

If new objections emerge, create files in:

- `knowledge/objections/`

If definitions change, update or create files in:

- `knowledge/concepts/`

## Workflow

1. **Identify the session trigger**
   - What started the conversation?
   - Was it a user insight, an external objection, a paper, a practical example, or a technical concern?

2. **Extract new concepts**
   - Name each concept.
   - Define it briefly.
   - Link it to existing concepts.

3. **Extract new hypotheses**
   - Convert ideas into testable or criticizable claims.
   - Give each hypothesis a stable name and ID if it deserves its own file.

4. **Extract objections**
   - Do not soften objections.
   - Write them in their strongest form.

5. **Record refinements**
   - Which previous idea changed?
   - Did a concept become narrower, broader, or more precise?

6. **Record open questions**
   - What remains unresolved?
   - What requires literature review?
   - What requires examples or empirical evidence?

7. **Identify repository updates**
   - List all files created or updated.

## Required sections for a distillation

Use this structure unless there is a strong reason not to:

```md
# Session Distillation — [Name]

Date: YYYY-MM-DD

## Context

## New concepts

## New hypotheses

## Important refinements

## Objections raised

## Open questions

## Files to create or update
```

## Style

Be concise but complete. The purpose is not to reproduce the conversation. The purpose is to preserve the knowledge that survived it.
