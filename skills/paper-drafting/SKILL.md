# Paper Drafting Skill

## Purpose

Use this skill only when converting mature parts of the theory into academic-style drafts, working papers, essays, or public-facing articles.

## Core warning

Do not write a paper too early. If the concepts, hypotheses, objections, and literature review are unstable, return to theory-building.

## Preconditions

Before drafting a paper section, verify that:

- the relevant concepts are defined;
- the relevant hypotheses exist;
- major objections are documented;
- related literature has been reviewed or at least mapped;
- open questions are clearly marked.

## Paper-writing principles

1. **State the research question clearly.**
2. **Distinguish thesis from speculation.**
3. **Do not overclaim originality.**
4. **Acknowledge limitations.**
5. **Use functional language instead of ideological labels.**
6. **Cite literature when making academic claims.**
7. **Keep the architecture coherent.**

## Standard structure

Use this default structure unless the target venue requires something else:

```md
# Title

## Abstract

## 1. Introduction

## 2. Problem Statement

## 3. Related Literature

## 4. Conceptual Framework

## 5. Functional Typology

## 6. Application to State Functions

## 7. Distributed Trust and Accountability

## 8. Objections and Limitations

## 9. Conclusion
```

## Drafting workflow

1. Select the smallest paper-worthy claim.
2. Identify which hypotheses support it.
3. Identify which objections must be addressed.
4. Identify which literature is required.
5. Draft only that section.
6. Mark unsupported claims explicitly.
7. Return unresolved gaps to `knowledge/open-questions/`.

## Style

Write with academic restraint. Avoid manifesto language. The paper should sound like a framework being proposed for critique, not a doctrine being announced.
