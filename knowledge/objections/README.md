# Objections

This folder stores objections that must be taken seriously.

The current Phase 3 working queue lives in `../../attacks/`:

- `../../attacks/README.md`
- `../../attacks/A001-legitimacy-does-not-follow-from-funding.md` through `../../attacks/A018-collusion-among-project-roles.md`

Those files are attack briefs, not accepted resolutions. Each should later become a resolved objection, contradiction resolution, accepted risk, deferred item, or explicit unresolved objection.

## Current major objection families

1. **Legitimacy:** Why would people accept decisions from distributed institutions?
2. **Conflicting authorities:** What happens when different arbiters or institutions disagree?
3. **Information capture:** Who controls the information layer?
4. **Popular bias:** How do we avoid funding only emotionally attractive projects?
5. **Neglected causes:** What happens to valuable but unpopular services?
6. **Use of force:** Where does distribution become dangerous?
7. **Transition:** How could a society move from the current model to a distributed one?
8. **Security and abuse:** How do we prevent fraud, manipulation, and coordinated attacks?
9. **Continuity:** How do essential services continue if funding becomes volatile?

Each objection should eventually become its own file.
