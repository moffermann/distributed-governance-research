# H012 — Distributed Value and Antivalue Management

## Hypothesis

Projects in a distributed governance architecture should be evaluated not only by the value they promise to generate, but also by the antivalue they create or may create.

## Definitions

**Value** is any positive social contribution perceived or validated by society: employment, education, health, dignity, connectivity, knowledge, care, cultural enrichment, security, opportunity, or reduced suffering.

**Antivalue** is any negative social effect that reduces the net value of a project: pollution, noise, health damage, environmental degradation, risk, abuse, fraud, social harm, displacement, or quality-of-life loss.

Antivalue is closely related to negative externalities, but the term is broader because it connects directly to the theory's concept of social value.

## Core idea

A project is not a demon simply because antivalue appears during execution. Significant projects often create tradeoffs. The key is whether the antivalue was declared, whether it can be mitigated, and whether society still perceives the project as producing net value.

## Net social value

A project can be understood as producing:

```text
social value generated - social antivalue generated = perceived net social value
```

The system should not require a central authority to calculate this perfectly. It should make value and antivalue visible so society can evaluate the tradeoff.

## Project obligations

A project should declare:

- its primary value proposition;
- secondary values;
- known or reasonably foreseeable antivalue;
- affected groups;
- mitigation plans, if applicable;
- KPIs and evidence standards.

## When antivalue appears

If antivalue appears during execution, the project does not necessarily lose legitimacy. The system should allow:

- community alerts;
- visibility of the conflict;
- mitigation projects;
- compensation projects;
- redesign projects;
- temporary pause;
- re-evaluation of social support.

## Mitigation projects

A mitigation project is a project whose primary goal is to reduce the antivalue generated by another project.

Example:

```text
Original project: build a highway.
Value: connectivity and lower travel times.
Antivalue: noise and pollution for nearby residents.
Mitigation projects: acoustic barriers, green buffers, relocation support, redesign of access points.
```

## Principle

> A project does not lose legitimacy automatically because it generates antivalue. It loses legitimacy if it hides antivalue, refuses mitigation, transfers undeclared costs to society, or if society concludes that its antivalue exceeds its value.

## Status

Core concept for project evaluation, externalities, mitigation, and social legitimacy.
