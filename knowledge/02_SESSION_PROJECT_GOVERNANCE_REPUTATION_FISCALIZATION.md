# Session Distillation — Project Governance, Antivalue, Reputation, and Fiscalization

Date: 2026-06-26

## Context

This session continued from the distributed planning architecture discussion and focused on what happens when projects generate conflicts, externalities, failures, or disputed value.

The discussion moved from high-level planning into project-level governance: how projects are paused, mitigated, revoked, evaluated, fiscalized, and reputationally tracked.

## New concepts

### Antivalue

A project can generate value and antivalue simultaneously. Antivalue is the negative social effect of a project: pollution, noise, harm, fraud, abuse, displacement, risk, or quality-of-life loss.

The system should evaluate projects by net social value, not only promised value.

### Mitigation project

A mitigation project is a project whose Primary Responsibility Anchor is to reduce the antivalue generated by another project.

This allows the system to avoid a binary choice between approving and cancelling projects.

### Fiscalization as a project

Fiscalization can itself become a distributed project. A community affected by a project can support a trusted fiscalizer to measure whether commitments and mitigation measures are actually fulfilled.

### Role-based reputation

Reputation should not attach only to institutions. It should also follow the people responsible for project design, execution, finance, auditing, and certification.

### Observability-based evaluation

Only actors who can observe or verify a dimension should formally evaluate that dimension.

## Key conclusions

### 1. Society defines value and antivalue

The system should not create a central authority that decides what counts as value or which harms are protected from social judgment. Society is sovereign in assigning value, but the system must expose information, evidence, and externalities.

### 2. Projects are responsible for undeclared antivalue

A project asking for resources must declare foreseeable antivalue. If relevant antivalue appears later and was not declared, the project should face responsibility mechanisms even if bad faith is not proven.

### 3. Pause is not revocation

If a project generates relevant antivalue, it may be paused while mitigation is designed, financed, executed, and verified. Revocation is more final and should occur only when mitigation fails, trust is lost, or society no longer perceives net value.

### 4. Mitigation creates new project opportunities

Conflicts should not be resolved only by political closure. They can generate new projects: mitigation, compensation, redesign, relocation, correction, or reconversion.

### 5. Financial guarantees matter

Transparency and reputation are not enough. The system needs guarantees, performance bonds, retentions, milestone-based disbursement, insurance, or escrow-like custody.

### 6. Fiscalizers must also be fiscalized

There is no perfect fiscalizer. Trust comes from standards, accreditation, conflict-of-interest disclosure, methodology, evidence traceability, peer review, second opinions, reputation, and sanctions.

### 7. Reputation must be recoverable but not erasable

Failures should affect reputation, but not necessarily permanently. Reputation should decay over time, allowing recovery. Fraud and concealment should weigh more heavily and decay more slowly.

## Open questions

1. How should the relevant voting community be defined for pause and revocation?
2. What threshold activates pause?
3. What threshold activates revocation?
4. What guarantee level is proportional to project risk?
5. Who accredits fiscalizers without creating a new monopoly?
6. How should reputation scores be computed without becoming opaque or gameable?
7. How should malicious complaints and sabotage be prevented?
8. When should fiscalization itself require guarantees?

## Research significance

This session transformed the project layer of the theory. The architecture now includes not only project funding and performance dashboards, but also antivalue declaration, pause, mitigation, revocation, role-based reputation, fiscalization markets, and responsibility for undeclared externalities.
