# P007 — Contextual Depth, Not Hard User Modes

## Principle

The system should not force citizens into hard interface labels such as basic user, advanced user, expert mode, or professional panel.

A citizen should be able to explore deeper functionality gradually, in context, as their interest in a specific project, category, territory, or process increases.

## Core formulation

> The citizen is not an advanced user globally. The citizen goes deeper locally.

## Rationale

A hard advanced-mode design creates unnecessary friction. It asks users to classify themselves before they know what they need.

The same citizen may behave differently in different contexts:

- basic participant in one project;
- interested observer in another;
- evidence producer in a third;
- active complainant in another;
- delegate or modeler in a topic they understand well.

Therefore, the interface should not divide people into permanent user types. It should reveal deeper actions when the citizen expresses more interest or takes a more complex action.

## Progressive disclosure model

The default layer should be simple.

Deeper layers should appear through natural exploration:

```text
project card
→ citizen project sheet
→ budget details
→ milestones
→ evidence
→ fiscalization
→ objections and complaints
→ audit trail
```

The user remains in the same product experience. They are not moved into a separate advanced panel.

## Design implications

The application should support:

- expandable sections;
- contextual action buttons;
- optional detail layers;
- guided explanations;
- AI assistance;
- progressive forms;
- task-specific depth;
- reversible exploration;
- simple defaults;
- full transparency for those who want to inspect deeper.

## Boundary

This does not mean that all users can perform all actions without eligibility rules.

Some actions may still require specific conditions, such as identity, reputation, authorization, absence of conflict, or technical requirements.

But access to information and progressive exploration should not depend on a global advanced-user label.

## Status

Accepted as a core usability principle.
