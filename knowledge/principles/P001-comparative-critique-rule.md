# P001 — Comparative Critique Rule

## Principle

Every critique of the distributed governance model must be compared explicitly with how the current institutional model solves the same problem. ^r27fc82cd

## Rationale

A common evaluation error is to demand perfection from a new model while accepting unresolved weaknesses in the current model as normal.

A difficulty is not automatically a refutation. It may be a general governance problem that both the current model and the proposed model must address.

## Test

When an objection is raised, ask:

1. Does the current system solve this problem?
2. If so, how?
3. Is that solution transparent, accountable, and effective?
4. Would the distributed architecture solve it worse, equally, or better?
5. What tradeoff changes?

## Example

Objection:

> What happens if two approved planning goals conflict?

Comparative response:

The current system also faces conflicting goals, such as reducing emissions while expanding mining. These conflicts are often resolved through political discretion, lobbying, budget negotiations, or executive preference.

The distributed model does not need to eliminate all conflict to be viable. It must show whether it can make the conflict more visible, more deliberative, more accountable, and more open to innovation than the current model.

## Status

Methodological principle for evaluating objections.
