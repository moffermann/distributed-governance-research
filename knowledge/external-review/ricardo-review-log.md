# External Review Log — Ricardo

## Purpose

This file tracks questions and objections raised by Ricardo, who functioned as an early external reviewer of the distributed governance theory.

The goal is not to reproduce the social media conversation, but to preserve the research value of the objections.

---

## R001 — Who writes the algorithm?

### Question / objection

If a system coordinates allocation, someone still writes the algorithm. Does that not recreate a political decision?

### Response developed

The key distinction is between an algorithm as decider and an algorithm as coordinator.

A decider algorithm chooses outcomes. A coordinator algorithm records, structures, aggregates, and executes decisions made by people.

### Resulting concept

- `knowledge/concepts/algorithm-as-coordinator.md`

## R002 — What does it mean to generate value?

### Question / objection

How is value defined, especially for things that are difficult to measure economically?

### Response developed

Value is broader than economic return. A school educates better, a care home cares better, a hospital saves lives, a foundation reduces suffering, a cultural center strengthens a community.

The system should not rely on a central authority or algorithm to define value. It should allow society to express value through distributed decisions and signals.

### Resulting files

- `knowledge/hypotheses/H002-distributed-resource-allocation.md`
- `knowledge/principles/P002-social-sovereignty-over-value.md`

## R003 — What about children or people who cannot decide for themselves?

### Question / objection

If value is expressed by people, what happens with actors who cannot directly express preferences, such as children?

### Response developed

The current system also represents children through parents, guardians, courts, ministries, and institutions. Distributed governance does not eliminate representation. It asks whether representation can become more plural, transparent, auditable, and contestable.

### Resulting concept

- Representation in `knowledge/concepts/core-glossary-v0.md`

## R004 — Who controls the information?

### Question / objection

If the system relies on information, who guarantees that information is truthful and not captured?

### Response developed

The answer is not a new central information authority. The answer is distributed trust: multiple sources, traceability, evidence, verification, auditability, reputation, and the ability to challenge claims.

### Resulting files

- `knowledge/hypotheses/H003-distributed-trust.md`
- `knowledge/concepts/distributed-trust.md`

## R005 — How do we avoid popularity contests?

### Question / objection

If people decide directly, does the system become a popularity contest?

### Response developed

The current democratic system also contains popularity dynamics. A distributed system can improve on this by adding assisted deliberation and performance dashboards.

The model becomes a form of augmented democracy: more direct, more informed, more traceable, and continuously evaluated.

### Resulting files

- `knowledge/hypotheses/H006-assisted-deliberation.md`
- `knowledge/hypotheses/H007-continuous-performance-evaluation.md`

---

## R006 — How are results verified?

### Question / objection

Projects can promise results and then report their own success. How does the system verify outcomes?

### Response developed

Accountability cannot depend only on self-reporting. It must include beneficiaries, users, affected parties, auditors, experts, evidence, financial traceability, complaints, and AI-assisted inconsistency detection.

### Resulting file

- `knowledge/hypotheses/H008-distributed-accountability.md`

---

## R007 — What happens when institutions concentrate power?

### Question / objection

Both sides are concerned with concentration of power, but what mechanism reduces it?

### Response developed

The theory distinguishes institutions from institutional monopolies. It does not necessarily eliminate institutions; it challenges monopoly institutions where distribution, competition, cooperation, and auditability can improve accountability.

### Resulting files

- `knowledge/hypotheses/H004-institutional-monopolies.md`
- `knowledge/concepts/institutional-taxonomy.md`

## R008 — How do we compare the new model with the current system?

### Question / objection

Many objections to the distributed model are real, but the current system also fails in those areas.

### Response developed

Every critique of the distributed model must be compared against how the current institutional model solves the same problem.
